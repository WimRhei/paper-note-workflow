# 算法论文 Markdown 模板

这份模板只规定 Algorithm 笔记的正文结构。共同的 Inbox 布局、文件命名、图表命名、嵌入和清理规则以 `../SKILL.md` 的 Output Contract 为准，不在这里重复。

只写原文支持的信息；缺失但重要的信息写 `未报告` 或 `FIX: info`。

## 模板正文

````markdown
# xxx

## 论文信息

- 标题：
- 作者与机构：
- 年份与会议/期刊/ArXiv：
- 链接：

> [!Key]
> 用 2-4 句话写清楚：论文解决什么任务，核心方法是什么，推理时真实执行链路是什么。只写原文可支持的信息，不写额外启发或推测。

## 研究对象与任务形态

写清楚论文研究的任务、输入、输出和调用方式。不要写成通用背景介绍，只保留理解推理链路所需的信息。

必要时用表格记录 workload：

| 项 | 原文信息 |
|---|---|
| 输入模态 |  |
| 输入长度 / 分辨率 / 帧数 |  |
| 上下文窗口 |  |
| 输出类型 |  |
| 输出长度 / horizon / chunk size |  |
| 调用形态 |  |

## 模型结构

写清楚主干模型、附加模块，以及这些模块在推理时分别承担什么角色。

| 模块 | 作用 | 原文给出的规模 / 配置 |
|---|---|---|
| 主干模型 |  |  |
| Encoder / Tokenizer |  |  |
| Decoder / Head |  |  |
| 其他模块 |  |  |

补充记录：

- 参数量 / 层数 / hidden size / head 数：
- dense / sparse / MoE / multi-stream / cascade：
- token 或 latent 类型：
- 推理时保留的 state：

## 推理 Workflow

先看不考虑优化时，一次推理到底怎么走。按真实执行顺序写，不按论文 marketing 名词堆模块。

```text
输入
  -> ...
  -> ...
  -> 输出
```

如果有循环、采样、搜索或多轮交互，写清楚：

- 循环类型：
- 每轮调用的模块：
- 调用次数 / step 数：
- 串行依赖：
- 可并行或异步部分：

## 关键机制

按论文真实机制组织。每个机制都写成：

- 机制解决的问题：
- 具体做法：
- 影响的推理阶段：
- 原文给出的代价或收益：

如果机制本身是训练方法，只记录它如何改变推理时的结构、调用次数、state、质量或效率；否则压缩处理训练细节。
````
